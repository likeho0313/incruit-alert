# 🚀 Wanted 채용 공고 디스코드 알림 봇

원하는 키워드의 채용 공고가 **원티드(Wanted)**에 새로 올라오면 실시간으로 **디스코드 알림**을 보내주는 자동화 봇입니다. GitHub Actions를 사용하여 24시간 내내 무료로 가동됩니다.

## ✨ 주요 기능
- **멀티 키워드 모니터링**: 여러 개의 키워드를 리스트로 관리하여 한 번에 검색합니다.
- **중복 알림 방지**: 이미 알림을 보낸 공고는 `last_jobs.json`에 저장하여 다시 알리지 않습니다.
- **완전 무료**: GitHub Actions를 사용하여 별도의 서버 비용 없이 작동합니다.
- **자동 스케줄링**: 기본적으로 1시간마다 새로운 공고를 체크합니다.

## 🛠 설치 및 설정 방법 (Setup)

### 1. 디스코드 웹훅 생성
1. 알림을 받을 디스코드 채널 설정 -> **연동** -> **웹훅** 메뉴로 이동합니다.
2. **새 웹훅**을 만들고 **웹훅 URL**을 복사해 둡니다.

### 2. GitHub Secrets 등록
보안을 위해 웹훅 URL은 저장소 설정에 별도로 저장합니다.
1. 본 저장소의 **Settings** > **Secrets and variables** > **Actions** 메뉴로 이동합니다.
2. **New repository secret** 버튼을 클릭합니다.
3. Name: `DISCORD_WEBHOOK_URL`
4. Value: 위에서 복사한 **디스코드 웹훅 URL**을 붙여넣습니다.

### 3. 워크플로우 권한 설정 (필수)
봇이 중복 방지 파일을 업데이트할 수 있도록 권한을 부여해야 합니다.
1. **Settings** > **Actions** > **General** 메뉴로 이동합니다.
2. 맨 아래 **Workflow permissions**에서 **Read and write permissions**를 선택하고 **Save**를 누릅니다.

## ⚙️ 개인 맞춤 설정 (Customization)

### 키워드 수정
`job_bot.py` 파일 상단의 `KEYWORDS` 리스트를 수정하여 원하는 키워드를 추가하거나 삭제할 수 있습니다.
```python
KEYWORDS = ["파이썬", "Python", "백엔드", "신입", "데이터 엔지니어"]
```

### 실행 주기 수정
`.github/workflows/main.yml` 파일의 `cron` 설정을 변경하여 알림 주기를 조절할 수 있습니다. (기본값: 1시간)
```yaml
on:
  schedule:
    - cron: '0 * * * *' # 매 정각마다 실행
```

## 📁 파일 구조
- `job_bot.py`: 원티드 API에서 공고를 가져오고 디스코드에 전송하는 핵심 로직
- `last_jobs.json`: 중복 알림을 방지하기 위해 이전에 전송한 공고 ID를 저장하는 파일
- `requirements.txt`: 실행에 필요한 Python 라이브러리 (requests)
- `.github/workflows/main.yml`: GitHub Actions 자동화 설정 파일

## 📝 로그 확인하기
작동 여부는 상단의 **Actions** 탭에서 확인할 수 있습니다.
- **Success**: 정상적으로 실행됨
- **Failure**: 로그를 클릭하여 에러 원인 확인 (웹훅 URL 확인 등)
